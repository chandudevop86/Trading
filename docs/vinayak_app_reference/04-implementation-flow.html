04-IMPLEMENTATION-FLOW

Ordered rollout steps for a Roboshop-style 3-tier implementation of Vinayak.

Step 1

Prepare the server or container runtime. Provision ECS, EC2, or a local Linux host. Open only web ports publicly and keep app and data layers private.

Step 2

Implement the web tier. Install Nginx or configure ALB. Point browser traffic to the Vinayak application and verify /admin and /workspace.

Step 3

Implement the app tier. Install Python dependencies, start FastAPI, validate health endpoints, and validate the candles and live-analysis APIs.

Step 4

Implement the data tier. Configure PostgreSQL or RDS, Redis or ElastiCache, S3 bucket for reports, and Secrets Manager for credentials.

Step 5

Integrate production controls. Enable CloudWatch logs, add CI/CD, run migrations during deploy, and verify report artifact creation and cache usage.

Step 6

Final production validation. Confirm browser traffic uses only the web tier, app tier stays private, data tier stays private, and live analysis works with Redis and S3 enabled.
