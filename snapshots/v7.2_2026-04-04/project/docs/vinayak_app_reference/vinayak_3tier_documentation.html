Vinayak 3-Tier Documentation

Communication flow, dependency split, and production-aligned explanation for the Vinayak trading platform.

Below is the communication between Vinayak components and their dependency flow. These dependencies come from the application design and current migration work. The infrastructure team can deploy them separately, but the functional dependency order stays the same.

Web Tier

The web tier handles browser-facing pages and public entry traffic. In Vinayak, this includes the admin and workspace pages, plus reverse-proxy or ALB ingress.

  • User entry through /admin and /workspace
  • Nginx or ALB acts as the frontend gateway
  • Traffic is forwarded only to the app tier

App Tier

The app tier contains the FastAPI layer, strategy processing, execution logic, live OHLCV services, Telegram flow, and report generation.

  • Dashboard APIs for candles and live analysis
  • Trading workflow orchestration
  • Execution and notification integrations

Data Tier

The data tier stores persistent data, cached market state, and generated artifacts.

  • Relational data through the DB layer
  • Redis for hot cache and latest analysis artifacts
  • S3 or local report storage for summaries and exports
Vinayak now satisfies the 3-tier logic in structure. The remaining work is environment validation on AWS and full production deployment verification.
