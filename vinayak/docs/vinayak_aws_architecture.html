1. Current Starting Point

Current Project Reality

  • Streamlit-based trading workspace UI
  • Python strategy and execution logic in the same application
  • Dhan broker integration and Telegram notifications
  • NSE and market-data fetch integrations
  • Nginx and Docker deployment guidance already present

Why AWS Fits

  • Easy path from a single EC2 app to multi-tier architecture
  • Natural use of RDS, ElastiCache, S3, ALB, Route 53, IAM
  • Good interview story: monolith to cloud-native design
  • Strong base for DevOps, cloud, and solution-architecture discussions

2. Target AWS Architecture

Public entry should stop at the web tier. The app tier should sit in private subnets, with the database and cache fully private.

4. Migration Stages

Stage 1: Lift and Shift

  • Deploy current app on EC2
  • Put Nginx in front of Streamlit
  • Attach Route 53 domain and ACM certificate

Stage 2: Separate App Logic

  • Move business logic into `vinayak/api` and `vinayak/strategies`
  • Add FastAPI endpoints and health checks
  • Keep UI and service logic clearly separated

Stage 3: Persistent Data

  • Introduce PostgreSQL schema for signals and executions
  • Move critical state from session/file to database
  • Add S3 for reports and archives

Stage 4: Performance and Async

  • Add Redis for hot-cache data
  • Add queue-based processing for notifications and execution requests
  • Reduce blocking behavior in live workflows

5. Security Design

  • Public internet access should terminate at ALB or Nginx only.
  • App tier should run in private subnets and talk to RDS/Redis privately.
  • Dhan, Telegram, and DB secrets should live in Secrets Manager or SSM Parameter Store.
  • Use IAM roles for EC2/ECS instead of hardcoding AWS credentials.
  • Security groups should tightly allow only required tier-to-tier traffic.
Interview line: "I would not expose the app tier or DB tier directly to the internet. Only the web tier should be public, while internal services stay private inside the VPC."

6. Resume and Interview Positioning

Present the project as a realistic migration path:

  1. Started with a Python trading monolith.
  2. Added a reverse-proxied web tier.
  3. Designed a target AWS multi-tier architecture.
  4. Planned the move to RDS, Redis, queues, and containerized deployment.
